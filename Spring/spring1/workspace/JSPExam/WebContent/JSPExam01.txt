	01.정적페이지와 동적페이지의 차이
		정적페이지는 저장된 페이지를 그대로 불러와서 보여줌(html)
		동적페이지는 요청에 따라 다른 정보를 제공(jsp, asp, php)
		
		정적페이지는 만들어진 페이지만 보여 줄 수 있으므로 고객의 취향이나 변화에 적응 할 수 없고,
		새로운 것을 추가 삭제 수정 하는 작업을 모두 수작업으로 해야하기 떄문에 관리가 어렵다.
		하지만 정적페이지의 경우 db구축이 필요없어 단순 문서로만 이루어져 있어 통신이 빠르다
		또한 정적문서로만 이루어져 있기 때문에 호스팅 서버에서도 동작이 가능하다는 장점이 있다.
		
		현재 우리가 보는 대부분의 웹페이지는 동적페이지임
		
		동적페이즈는 사용자가 웹페이지에 글을 작성하거나 환경설정 등을 바꾸면 그 내용이 서버에 있는 db에 저장되고 결과가 웹페이지에 반영되는 형태
	
	02. JSP는 서블릿 기술의 확장
	 
	 	JSP에서는 서블릿의 모든 기능을 사용할 수 있음
	 	즉 서블릿과 마찬가지로 JDBC, JNDI, EJB 등을 모두 포함하고 있는 강력한 자바 api를 사용ㅎ가능
	 	또한 내장 객체, 미리 정의된 태그, 표현식 언어와 사용자 정의 태그를 모두 사용가능
	 	
	 	JNDE?
	 	Java Naming and Directory Interface로 java프로그래밍 언어를 사용하여 작성된 에플리케이션에
	 	이름지정 및 디렉토리 기능을 제공하는 api
	 	
	 	EJB?
	 	Enterprise java bean: 분산 애플리케이션을 지원하는 컴포넌트 기반의 규약
	 	
	 	JSP는 유지 관리가 용이
	 	
	 	JSP는 빠른 개발이 가능
	 	
	 	JSP로 개발하면 코드 길이를 줄일 수 있음
	 	
	 		다양한 액션태그, jstlm 사용자 정의 태그 등 다양한 태그와 표현언어, 내장객체 등을 사용함으로 써 서블릿보다 코드를 줄일 수 있음
